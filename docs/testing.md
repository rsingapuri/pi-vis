# Testing

## Verification commands

```sh
npm run typecheck
npm run lint
npm test
npm run test:render
npm run test:e2e
npm run test:e2e:real-pi   # focused pinned-Pi compatibility journeys
npm run test:transcript    # focused transcript/history/tool-result reducers
npm run test:full          # typecheck, lint, unit, build, render, Electron E2E
```

`npm run test:full` is the local release gate. If the non-TTY lint wrapper reports a parser artifact, verify with `./node_modules/.bin/biome check .` or `script -q /dev/null npm run lint`.

## State-authority regression coverage

- `resources/pi-session-host/state-authority.test.mjs` is the direct fault-injection gate. It tests direct snapshots, identity/epoch transitions, FIFO custody behind compaction/navigation, callback-time compaction getter reconciliation, submission dispositions, ESC target selection, queue-restoration attachment custody, strict all-owned untransformed/attachment-free queue remove/edit/reorder rebuilds, editor revision rejection, and transition batches.
- `src/main/pi/session-host.test.ts` exercises host-envelope identity/sequence validation, transport-gap fencing, full-snapshot recovery, host control watchdogs (including the sleep/wake suspension gap that must not retire a healthy host), and host wire acknowledgements using the stub-fork seam.
- `src/main/sessions/session-registry.test.ts` covers snapshot leases/availability, uncapped activation, untouched activation-visit retirement and its safety fences, retained-intent uncertainty after failure, stale and lost-ack ESC, unified-submit continuation identity, renderer-generation claims, bounded claim expiry/tombstones, acknowledged renderer/UI operations, close prepare/confirm mutation checks, and correlated/ambiguous reload settlement.
- `src/main/sessions/bound-history.test.ts` fault-injects cold activation, same-file host restart, epoch transition, file mismatch, and close/recreate while persisted history I/O is delayed.
- `src/renderer/src/stores/sessions-store.test.ts` verifies renderer snapshot ordering/availability gating, dispositions, queue restoration, editor revisions, unified host/epoch response binding, expired continuation suppression, same-file/same-cursor delayed-history fencing, generation-safe UI projection, authoritative user-echo attachment rendering, and terminal background-turn done/error notifications.
- `src/renderer/src/components/composer/Composer.autocomplete.test.tsx` and `src/renderer/src/stores/tree-store.test.ts` enforce unconditional `/tree` overlay admission during compaction/abort authority fences, retryable unavailable opening, idle-only branch mutation, first-user-echo clearing during pending-session promotion, the frozen "sending" composer state (disabled + spinner until acceptance or failure), and the toast-free transient denial of prompt submission during compaction.
- `resources/pi-session-host/bridge.test.mjs` additionally pins that a submit dispatched mid-compaction reaches custody immediately (deferred intent settlement keeps the serialized ingress scheduler free), and `src/renderer/src/components/session-header/SessionHeader.changes-button.test.tsx` pins changes-button error presentation plus refresh after a tool completion on the authority transcript plane. `sessions-store.test.ts` also covers `reactivateSession` recovery for a failed active session.
- `tests/fake-session-host-esc.test.mts` drives the real child-process fixture over IPC and verifies navigation → compaction → retry → streaming → bash → editor ESC priority, cancellation, late-output/persistence suppression, and attachment-aware queue restoration.
- `tests/lifecycle-boundary-model.test.mts` exhaustively enumerates the finite internal operation/boundary/lifecycle-cut reference model for submissions, effectful commands, reload, ESC, editor patches, picker continuations, and claimed/unclaimed unified actions, then runs reproducible generated schedules over the same state space. The model explicitly checks one dispatch per intent, terminal-versus-review exclusivity, permanent `outcome_unknown` replay refusal, correlated Composer non-clearing after a lifecycle cut, and zero admission while unavailable, transitioning, closing, or identity-stale. It is not treated as implementation conformance by itself: deferred-promise tests in `bridge.test.mjs`, `state-authority.test.mjs`, `session-registry.test.ts`, and `sessions-store.test.ts` drive the real adapters at those cuts (including active-command/replacement overlap, predecessor-epoch terminal submission, command settlement after epoch change, close freeze, stale picker/editor continuation, and unified expiry). The model is not a universal proof of arbitrary JavaScript schedules.

## Workspace session-search coverage

Search tests use a generated isolated JSONL corpus: two workspaces, a mapped worktree, more than 30 sessions, exact/prefix/camel/snake/path/typo cases, pre-compaction and alternate-branch-only matches, repeated occurrences, corrupt/intermediate rows, incomplete and appended rows, and an excluded archived source. Unit/integration coverage verifies catalog ownership/confinement, transactional append/rebuild/corruption/restart behavior, deterministic relevance (Recall@10, MRR, nDCG), generation fencing, revision-invalidated pagination, exact context relocation/stale outcomes, and opaque-target validation. Electron/render coverage proves scope and worktree inclusion, sidebar-pagination independence, default-on/restart-scoped enablement, disabled-launch chrome suppression, ESC/focus behavior, preview's zero host/session side effects, and explicit open's normal activation-visit release behavior.

Before release, run `PIVIS_SEARCH_BENCH_MIB=500 npm run bench:session-search`, build the `.app`, then run `node scripts/verify-packaged-session-search.mjs <path-to-Pi-Vis.app>`. The packaged check launches the asar-unpacked worker under the packaged Electron Node runtime and requires SQLite/FTS initialization. The benchmark drives the production service, catalog, and compiled worker against an isolated corpus, samples process RSS from the unblocked parent during indexing, and gates relevance, cold progressive first result, warm-query p95, and end-to-end completed-append detection from an early source while a bulk source is still reconciling. Gates: warm query p95 <150 ms; first initial-index result <500 ms; complete append <2 s; stale UI suppression <100 ms; no search-caused main-thread task >16 ms; worker RSS <192 MiB for 500 MiB sources; IPC batches <=50 results and <=128 KiB; preview creates no host.

Conversation-tree copy changes also run `npx vitest run src/renderer/src/components/tree/tree-flatten.test.ts src/renderer/src/components/tree/tree-copy.test.ts` and `npm run test:render -- tree-copy.spec.mts`; the render flow pins the exact clipboard payload plus input/native-selection non-interference.

Model-catalog SWR coverage must prove that `refreshModels` emits no catalog payload, only a same-owner terminal outcome permits the follow-up `get_available_models` read, failures preserve the cache and expose retry only for that owner, and auth/picker triggers never activate a cold session. Runtime-login coverage additionally validates dynamic provider/method discovery, trust and method revalidation, cancellation, secret-free outcomes/replay snapshots, and persistent auth-surface revision replacement. `tests/render/provider-login.spec.mts` captures the provider chooser, an empty password-style API-key prompt, and OAuth device-code card; it submits a test secret, proves the DOM/Zustand projection does not retain it, exercises code copy/system-browser routing, and observes the post-login model catalog.

Run the focused tests while changing these surfaces:

## Other layers

Vitest covers TypeScript source, host `.mjs` units, test harness units, and deterministic child-process protocol tests. Playwright E2E builds a fresh production app; renderer tests run against the preview stub. Normal Electron E2E includes delayed history activation/retry, a 10–20 MiB large-session fixture whose bytes survive into rendered tool output, renderer heartbeat and main-IPC responsiveness gates during cooperative archive mounting, claimed-unified watchdog recovery with exactly-one dispatch, process-level streaming/queue/compaction/bash ESC cancellation, and a fake-host seam where the first authoritative user echo precedes the submit response so Composer echo deduplication is exercised in the real IPC ordering. `tests/fixtures/fake-pi.mjs` is retained for executable version and update tests, while `fake-host-process.mjs` drives the SDK-host wire protocol without a real pi install.

Extension-package update coverage is split between `src/main/extension-updates.test.ts` (user-scoped worker checks, launch/Settings single-flight caching and retry, packaged worker resolution, serialized mutation ownership, process-tree timeout escalation, and the invariant that runner arguments never contain pi self-update targets), `src/renderer/src/stores/extension-updates-store.test.ts` (renderer request coalescing and retry), and `tests/render/extension-updates.spec.mts` (automatic first-open checking, cached reopen, manual refresh, the Settings update-count badge, and extension-only update targets).

The normal Electron suite also runs the real SDK host with the exact pinned production dependency `@earendil-works/pi-coding-agent@0.80.10` — the same runtime the app bundles and ships. `tests/e2e/support/real-sdk-host.mts` resolves that repository-local executable, verifies its reported version, and creates isolated home, settings, agent, workspace, extension, and one-level session-bucket directories. It never searches `PATH` or falls back to a developer installation.

The real-Pi suites mock only the model provider with `tests/e2e/support/scripted-openai-provider.mts`, an ordered OpenAI-compatible server bound to `127.0.0.1`. They exercise the real `AgentSession`, extensions, tools, persistence, host IPC, authority frames, renderer store, and Electron DOM. Mandatory journeys cover Composer keyboard and attachment custody, an immediate first prompt using a configured default model (without a model-selection command warming attachment), transformed and duplicate prompts, multi-chunk streaming, extension notifications/status/widgets/dialogs/editor mutation/errors/custom entries (including a renderer that declines the default 80-column probe), settled tool cards and history parity, partial disconnect and terminal provider errors, automatic HTTP retry, real tool/retry/stream interruption, queued-message recovery, empty and successful manual compaction, post-compaction admission, and full process reload. Assertions include provider request history and persisted JSONL, so dispatch or Composer clearing alone never counts as success. `PIVIS_REAL_USER_HOST_SMOKE=1` remains an optional developer-resource smoke test; provider-spending and real-panel suites remain opt-in where documented.

`tests/e2e/real-sdk-regressions.spec.mts` is a mandatory focused pinned-Pi gate. It uses only `createRealSdkFixture` and the exact local 0.80.10 binary, throwaway state, a loopback scripted provider where model work is required, and a real extension fixture. It covers real factory/static widgets and unified/custom Escape custody, native command precedence and immediate gated compaction, session naming/tree/relaunch reconstruction, plus pending-draft recovery while saved-session search owns focus.

`npm run test:e2e:real-pi` runs the existing smoke, transcript-lifecycle, and regression suites together with `tests/e2e/real-sdk-races.spec.mts` and `tests/e2e/authority-faults.spec.mts`. The races suite covers `/new` and `/reload` attach/focus storms, admission-time `/compact` clearing during an attach race, and gated streaming close. The authority-faults suite exercises deterministic child-IPC fault and recovery journeys.

A pinned-Pi upgrade is a production runtime change (the app ships the pin): update the exact dependency and `PINNED_PI_VERSION`, audit and update the compatibility note, run `npm ci`, then run `npm run test:full` (or at minimum `npm run test:e2e:real-pi` plus the unit gates). CI uses Node 22.19.0 because Pi 0.80.10 declares Node `>=22.19.0`, and runs render and Electron E2E under `xvfb-run` without credentials.

Worktree commands use `scripts/ensure-worktree-dev.mjs` to reuse a compatible sibling `node_modules`; caches are worktree-local. E2E uses isolated settings/session directories and defaults to one worker.

## Authority-frame fault and invariant coverage

[ADR 0003](decisions/0003-authority-frames-and-plane-synchronization.md) is deployed and its matrix is covered at the child, main-router, reducer, and real-Electron layers. `resources/pi-session-host/state-authority.test.mjs`, `src/main/sessions/renderer-publication.test.ts`, `src/renderer/src/stores/authority-reducer.test.ts`, registry/store deferred-promise tests, and `tests/e2e/authority-faults.spec.mts` cover:

- exact-as-of-cursor `following` semantics, baseline plus contiguous replay (including renderer materialization of replayed transcript events), and dropped, reordered, duplicate, and bounded-buffer-overflow publications;
- attach racing replacement, same-owner concurrent/superseded attaches, baseline reads while mutation ingress is blocked, immediate transport-fence rejection, detached compaction/reattach, journal low/high-watermark and truncation behavior, and predecessor fencing;
- stable-intent deduplication/conflicting-ID rejection, lost dispatch receipts as `delivery_unknown`, and permanently non-replayable `outcome_unknown`;
- conservative getter/event barriers (including compaction retry), independently fenced transcript/extension-UI/panel planes, keyframe or acknowledged-repaint panel input fencing, advisory-lock contention, and child-loss non-success.

`PIVIS_TEST_FAULT_PLAN` enables deterministic per-`SessionHost` child-IPC faults. It is JSON only: a flat `rules`/`faults` array or `inbound`/`outbound` arrays; each rule matches `type`, optionally `plane` and `nth`, and applies `drop`, `duplicate`, `delay` (`delayMs`), or adjacent-message `reorder`. Missing or malformed plans are no-ops. Real-SDK fixtures clear inherited plans and set this variable only for the launched Electron process. `PIVIS_TEST_AUTHORITY_BUFFER_LIMIT` creates the bounded attach-buffer seam, and `PIVIS_TEST_IPC_INVOCATION_LOG` records invocation evidence used by fault tests. `PIVIS_TEST_REAL_HOST_CONTROL=1` additionally enables the real-Pi fixture's owner-bound reload replacement and active-child kill controls; main, host wrapper, and child each reject them without that exact flag, and control acknowledgements are written to the IPC log.

The standard and pinned real-SDK Electron suites use the strict invariant fixture in `tests/e2e/support/invariants.mts` and launch through `instrumented-launch.mts`. It fails a test on unallowlisted renderer console errors, page errors, native dialogs, main-process uncaught-handler/rejection stderr, or error toasts; an expected diagnostic must be scoped with `allowInvariant`. The real scripted provider supports deterministic latency at provider, step, and response scope. Real-SDK journeys use seeded 10–40 ms first-byte and 5–30 ms per-chunk delays, rather than treating an instant fake response as streaming coverage.

Electron launch checks that `out/main/index.js` and copied host resources are fresh against every source host `.mjs`; run `npm run build` when it fails. `PIVIS_TEST_SKIP_FRESHNESS=1` is the explicit test-only bypass.
