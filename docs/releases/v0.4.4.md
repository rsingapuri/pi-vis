# Pi-Vis v0.4.4

## Highlights

- Added inline editing to the diff viewer:
  - Select changed lines and choose **Edit selection** to update the working file without leaving the review.
  - The editor supports auto-indent, Tab/Shift-Tab indentation, syntax highlighting, and Cmd/Ctrl+Enter to save.
  - Unsaved edits are protected by a discard confirmation, and external file changes are detected before saving to prevent accidental overwrites.
- Improved search and navigation in large diffs:
  - Diff search now scans the complete change set, including files and lines beyond normal rendering limits.
  - Search and diff-model processing run off the renderer thread for a more responsive interface.
  - Large files render in bounded chunks instead of mounting oversized views at once.

## Fixes and polish

- Improved Escape/interrupt reliability during active runs by tracking whether the current operation can be interrupted.
- Added support for Pi's `agent_settled` event so streaming and interrupt state clear at the correct point after retries or continuations.
- Reduced diff refresh churn while editing and removed loading flashes after saving changes.
- Made the closed diff-view changes badge cheaper to update by avoiding unnecessary full-diff loading.
- Hardened working-file saves against path traversal, symlink targets, non-regular files, and conflicting on-disk changes.

## Tests and tooling

- Added unit, render, and E2E coverage for inline diff editing, save conflicts, large-diff search, rendering limits, and interrupt behavior.
- Documented the worker-backed diff search and safe working-file editing architecture.
