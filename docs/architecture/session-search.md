# Workspace session search

See [ADR 0002](../decisions/0002-workspace-session-search.md) for the binding invariants.

## Boundaries and authority

A renderer search modal calls a typed, generation-fenced main `SessionSearchService`. The service owns an asynchronous, metadata-only catalog and one prioritized worker for streaming JSONL extraction, SQLite FTS, ranking, and read-only context. One global index is keyed by owning workspace; every request is scoped to one currently registered workspace and its known worktrees. The index at `<userData>/session-search/v1/` is private, disposable, and rebuildable. Persisted JSONL is the sole content authority.

The catalog accepts only canonical regular `.jsonl` files under the canonical sessions root, validates headers, and revalidates workspace/worktree ownership for queries and context. Catalog, index, context, and explicit-open gates bind post-open containment and device/inode identity to the descriptor actually read, so an intermediate-directory symlink swap cannot escape the root. Header `cwd` is untrusted. Archived sources remain catalogued for classification but are excluded from initial search. Search is independent of sidebar expansion, pagination, and active workspace.

Searchable content is session names, user messages, visible assistant prose, persisted errors, displayed custom messages, compaction/branch summaries, pre-compaction history, and all persisted branches. It excludes thinking, ordinary tools, images/base64, hidden custom or arbitrary extension data, tool metadata paths, drafts, and unpersisted streaming output. A compaction marker is not a history boundary. “Latest persisted path” is deterministic persisted graph order, never a claim about the live active branch.

## Index lifecycle and failure

The long-lived worker gives queries priority over row- and byte-bounded indexing transactions. It persists the next query-visible index revision atomically in every committed chunk or metadata/graph mutation, before servicing priority queries, so pagination can detect ranking changes during progressive reconciliation or after worker restart. It indexes complete newline-terminated rows with source cursor advancement; an interrupted commit may repeat work but cannot permanently skip it. Duplicate entry identity is enforced by SQLite rather than an in-memory corpus-sized set, and latest-path classification is a recursive SQLite operation rather than a retained JS graph. Cold discovery iterates directories with `opendir`, yields fixed-size inspection batches without materializing/sorting whole directory entry arrays, and prioritizes directory names matching an open search's workspace; a search that starts during startup discovery dynamically promotes matching directories already queued as ordinary, without restarting traversal. If catalog traversal has already completed but its bulk worker reconciliation is still active, search submits one deduplicated priority reconciliation for that workspace's not-yet-reconciled sources. The first owned batch can begin progressive indexing before global enumeration completes. A warm traversal publishes the union of its prior complete snapshot and newly inspected batches, preserving concurrent append/removal observations, then atomically adopts the completed map so known results never disappear merely because their batch has not been revisited yet. Only genuinely new sources are sent through progressive incomplete reconciliation; unchanged warm batches wait for the single final complete reconcile. Concurrent known-source checks merge per-path deltas into the latest map, and a full-scan tombstone wins over delayed inspection of the same path. While search is open, a 250 ms stat-only pass fully re-inspects and reconciles only changed known sources. Startup performs complete catalog reconciliation. Modal search requests, app focus/resume, and the open-search timer request later full scans, but all ambient requests are coalesced and suppressed until 30 seconds after the last completed full scan, so query debounce, focus churn, or interval phase cannot duplicate a just-finished reconciliation. Explicit rebuild remains ungated; a successful rebuild records the same completion timestamp, while a failed rebuild does not delay ambient recovery. Worker reconciliations assign per-path request generations: when a nested priority append supersedes an older bulk descriptor, every resumed chunk/finalization checks the generation and the stale bulk path cannot overwrite the newer index state. Shrink, replacement, header/inode/prefix mismatch, or committed-content mismatch rebuilds a source.

Corrupt or incompatible indexes are quarantined and rebuilt; incomplete migrations are cleaned up; a worker crash has one bounded, reinitialized restart. Error/exit events are bound to the worker that emitted them so a stale exit cannot terminate its replacement; failure of the replacement becomes terminal for search instead of creating an uninitialized worker. Errors, permission failures, malformed rows, partial rows, deletion, and skipped sources are reported as incomplete coverage without blocking normal session use. The worker never modifies JSONL or retains complete multi-session transcripts in memory.

## Results, context, and opening

Results identify an opaque target, not an authoritative source path. Targets are renderer- and workspace-scoped. The worker groups by catalogued source path/session identity (never a display session name) before pagination, emitting one opaque representative per source and its `additionalMatches` count. A pagination offset belongs to one index revision and a fingerprint of its sorted pinned, expanded, and allowed-source inputs; if progressive indexing or those ranking/filter inputs change before the next page, main discards the shifted offset response and replaces the list with a fresh first page rather than emitting duplicates or omissions. A target resolves an exact entry, content part, and occurrence; snippets/highlights use original JSONL text. Repeated occurrences and context IPC are byte/count bounded. Multi-term highlight ranges remain one logical segment result. Late batches/context cannot affect a successor query or selection. If a replacement page removes the selected target, selection moves and its old context is synchronously cleared; the context pane additionally renders ready/error/loading state only when its target still equals the current selection.

Arrow selection only changes result state. Clicking a result or pressing Enter explicitly enters the read-only preview and only then asks the prioritized worker for bounded saved-history context after main revalidates authority. Context requests are serialized per search, the worker never nests one context scan inside another, and its total pending request queue is capped; long scans yield only to bounded query/status/exact-validation work. It revalidates source revision and ownership, relocates only by entry id/content-part key/occurrence/digest after an append, and returns `ready`, `relocated`, `changed`, `removed`, `forbidden`, or `unavailable`. Context graph work is capped at 100,000 entries and 32 MiB of retained metadata; a larger single source reports an honest unavailable preview rather than exhausting the worker. Missing ancestry is displayed as incomplete; changed content is never silently substituted. Other saved branches are explicitly labelled and opening does not navigate to that branch. Context never seeds live history or invokes extension rendering.

Opening the search surface, querying, and preview have no `SessionRegistry` or SDK-host interaction and create no host. **Open session** alone revalidates a target and delegates to the existing stored-session open path. Main opens and confines the source descriptor *before* worker validation; the worker rechecks the descriptor's header session id and exact catalogued `cwd` authority before checking the indexed row range, entry id, content-part key, and digest, even when size/mtime/inode/prefix metadata appears unchanged. Preview context uses the same descriptor-header gate, and only relocation read through that descriptor may recover a moved target. Main retains the validated descriptor across the cold renderer-activation IPC gap, including when adopting an existing cold record. At each POSIX spawn it rechecks pathname/inode identity, opens a same-inode fresh offset-zero `O_APPEND` description, and inherits that into the normal SDK host as child fd 4. Windows, which has no descriptor filesystem, creates a PID-tagged random non-JSONL hard link beside the source and accepts it only after its inode matches the retained descriptor; the SDK host uses that stable same-inode path, registry cleanup retries transient unlink failures, and catalog startup reaps pins whose owner process no longer exists. Thus runtime reads and appends remain pinned to validated content rather than a pathname checked earlier. The descriptor or runtime pin closes with the registry record. The shared renderer path selects the validated saved session independently of visit-triggered host startup: persisted history becomes the visible session immediately, while activation continues through the ordinary activation-visit lifecycle. After renderer adoption, explicit search open requests the ordinary authority attach for both `starting` and `ready` lifecycle snapshots. Main admits that handshake only after activation's main-owned lifecycle commit is complete; a successful serialized baseline can then safely repair only a stale local `starting` projection. Thus a `ready` event that crossed the adoption IPC gap cannot strand the session, a provisional child baseline cannot outrun bound-history ownership, and a newer terminal event remains fenced. If activation fails (for example because runtime model/auth availability changed), the selected session remains open with a retryable failed runtime instead of turning a valid search result into an open failure. It introduces no fixed active-host cap, generalized idle/LRU reaper, or retirement of pre-existing hosts.

## UI and performance

When launch-scoped search availability is enabled, each workspace row has a hover- or keyboard-focus-revealed search icon beside its collapse chevron; those workspace actions collapse to zero size at rest, and `Cmd/Ctrl+Shift+F` scopes the modal to the active workspace. `sessionSearchEnabled` is a default-on persisted escape hatch in Settings; main reads it at launch, and when disabled on the next restart it does not create the service, catalog, or worker. The renderer asks main for launch availability rather than treating an unsaved/current toggle value as runtime authority. The modal is an ESC-claiming, focus-trapping direct `.app` overlay. Its result selection changes only search state; narrow layout uses first ESC to return from context and second ESC to close. Close restores the invoking icon focus and never sends `session.escape`.

A generated isolated corpus and E2E coverage include multiple workspaces, mapped worktrees, >30 sessions, exact/prefix/component/typo cases, pre-compaction and alternate-branch matches, repeated occurrences, malformed/incomplete/appended rows, and archived exclusion. Relevance tracks Recall@10, MRR, and nDCG. Gates: warm p95 <150 ms; first initial-index result <500 ms; appended complete entry <2 s; stale suppression <100 ms; no search main-thread task >16 ms; worker RSS <192 MiB for 500 MiB sources; IPC batches <=50 results and <=128 KiB; preview creates no host.
