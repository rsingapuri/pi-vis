# Conversation tree

`/tree` opens the native conversation-DAG overlay and changes the active leaf in place; `/fork` creates a separate session. **Opening the overlay is an unconditional renderer-local invariant:** command classification happens before runtime/editor admission, and the Composer keeps accepting `/tree` while semantic authority is synchronizing or unavailable (including compaction, abort, startup, and host failure). The overlay opens immediately with a retryable unavailable state and refreshes when semantic authority follows again. Runtime-backed attachments and every non-local submission remain fenced. The SDK host uses pi's public `sessionManager.getTree`, `getLeafId`, `getBranch`, `appendLabelChange`, and `session.navigateTree` surfaces. Tree support is an SDK-host capability; when the installed pi lacks a required public method, the overlay reports it as unsupported without changing runtime mode.

## Wire format

`getTree()` is recursive and can exceed Electron contextBridge's nesting limit. The host therefore sends flat `{ entry, parentId, label?, labelTimestamp? }` nodes, so only shallow data crosses preload. The renderer reconstructs and traverses the nested presentation with explicit iterative stacks (`tree-flatten.ts`); session-controlled conversation depth must never become JavaScript call-stack depth.

After navigation, the returned in-memory root→leaf branch is converted by `entriesToTranscript`; Pi-Vis does not reread the session file because recently appended entries may not yet be persisted. Tree navigation is refused whenever the authoritative runtime snapshot is unavailable or its complete SDK `isIdle` fact is false, covering streaming, compaction, retry, bash, and other barriers without preventing the viewer itself from opening. The SDK host repeats the public `isIdle` check inside serialized navigate-intent execution immediately before `navigateTree`, closing the renderer-observation race with work admitted from another surface. Pi's public `isCompacting` getter also becomes true for branch summarization during `navigateTree`; while `runNavigation` owns that operation, authority masks the generic getter from the context-compaction reconciler. The renderer likewise recognizes context compaction only from the child-owned `activity.compaction` projection, never the raw SDK diagnostic bit. Navigation therefore cannot fabricate a compaction barrier or working indicator that lacks a matching `compaction_end`, and Escape continues to target navigation. Tree reads carry their captured cursor as observation context, but a same-owner semantic cursor advance does not stale their response; only loss of following authority or an owner/epoch replacement does. Successful navigation remains on that same owner: its matching terminal `navigate` outcome in a semantic frame carries the leaf and in-memory branch, while the terminal snapshot carries any revisioned editor change. The renderer converts that branch into a transcript presentation baseline only while the exact outcome and owner remain current, then resumes interaction.

The viewer mirrors the diff overlay's modal and Escape ownership conventions. Filtering is per-node, hidden ancestors reattach visible descendants, indentation follows actual visible branch points, and tool-result messages are identified from pi's assistant content. The selected row can be copied with its hover/focus `IconCopy` control or Cmd/Ctrl+C. Copy uses the existing `clipboard.writeText` IPC and extracts the full text from Pi's copyable message, custom-message, compaction, and branch-summary entries rather than their truncated one-line row summaries; bookkeeping-only rows stay disabled, and success is a transient check icon rather than a toast. The shortcut never takes ownership from an input, contenteditable, or native browser text selection, and it does not navigate or close the viewer.
