# Runtime services

Read this when changing browser preview, auth/login, embedded PTY, update checks/runs, subprocess environment, or native runtime dependencies.

- **Browser preview**: `npm run dev:renderer` loads `preview-stub.ts` which stubs `window.pivis` with a demo session and canned responses including streamed agent output.
- **Auth / subprocess env**: API keys stored in `~/.pi/agent/auth.json` using `proper-lockfile` for mutual exclusion with pi's token-refresh writes. Environment variables detected via `$SHELL -ilc env` (GUI apps don't inherit shell env). `getSubprocessEnv()` combines `process.env` + login-shell env for consistent subprocess PATH — used across git, updates, pty, and locate-pi. Pi session spawns additionally merge `settings.piEnv` (advanced Settings UI, one `KEY=value` per line; applies on new spawn or `/reload`) after the login-shell env and before Pi-Vis-owned `PIVIS_*` controls. Atomic writes with tmp+rename, chmod 0600.
- **Login**: Native API-key sign-in (writes auth.json) + embedded xterm.js terminal for OAuth (spawns real `pi` in `node-pty`, watches `auth.json` for success detection).
- **Pi / extension updates**: Background check at launch (3s delay, non-blocking, respects `updateCheckEnabled` setting). `update.run` spawns `pi update … --no-approve` via `spawn()` with 10-minute safety timeout, streams output via IPC events. The target maps to an explicit flag — `"all"` → `--all` (pi **and** extensions), `"pi"` → `--self`, `{extension}` → `--extension <src>`. This matters: bare `pi update` defaults to pi-only and **silently skips extensions** ("Run pi update --extensions to update extensions."), so the "Update now" button must pass `--all`. New sessions automatically use the updated binary.
- **Pi-Vis app updates**: Separate from pi/extension updates. `src/main/app-updates.ts` uses Electron's built-in `autoUpdater` only for packaged macOS builds (`app.isPackaged`, `darwin`, not `PIVIS_DISABLE_APP_UPDATES=1`). The feed defaults to `https://update.electronjs.org/rsingapuri/pi-vis/${process.platform}-${process.arch}/${app.getVersion()}` and can be overridden with `PIVIS_APP_UPDATE_FEED_URL` for diagnostics. Background check runs at launch after 5s and respects `appUpdateCheckEnabled`. IPC channels are `appUpdate.status`, `appUpdate.check`, and `appUpdate.install`; events are emitted as `appUpdate.status`. When Squirrel.Mac reports `update-downloaded`, the renderer shows a restart-to-install prompt and calls `autoUpdater.quitAndInstall()` through `appUpdate.install`.
- **Dependencies**: `@homebridge/node-pty-prebuilt-multiarch` (native, externalized from main bundle, asarUnpack in electron-builder), `@xterm/xterm` + `@xterm/addon-fit` (renderer), `proper-lockfile` + `@types/proper-lockfile` (main).
